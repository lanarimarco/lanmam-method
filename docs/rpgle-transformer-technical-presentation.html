Analysis Agent - The Program Detective
Purpose: Deep analysis of RPGLE source to extract business logic, data flows, and dependencies

Inputs

ğŸ¯ Prompt Engineering & Agent Design

Each Agent Has Structured PROMPT.md

PROMPT.md Structure
## Purpose - Clear statement of agent's role
## Inputs Required - Exact list of needed files/data
## Outputs to Produce - Explicit deliverables with paths
## Process - Step-by-step instructions
## Output Format - Templates and structure
## Quality Checklist - Validation before completion

LLM-Agnostic Design

Works with ANY LLM:

  • Claude (Opus, Sonnet)
  • GPT-4 / GPT-4 Turbo
  • Gemini
  • Local models (Llama, etc.)
  • GitHub Copilot

Prompts are self-contained markdown - copy/paste into any AI tool

Two Execution Modes

Mode 1: Single Session
Best for: Large context LLMs
Method: Use MASTER-PIPELINE-PROMPT.md
Pros: Faster, better context retention
Cons: Requires ~40KB+ context window
Mode 2: Phase-by-Phase
Best for: Cost control, smaller LLMs
Method: Run each PROMPT.md separately
Pros: Review between phases, flexible
Cons: More manual coordination
